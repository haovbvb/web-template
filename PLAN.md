# 企业级全栈模板阶段计划清单（单人开发版）

## 0. 项目约束与目标

- 开发模式：全程 1 人全栈（产品、前端、后端、运维由同一人承担）
- 部署方式：Docker / Docker Compose 为默认交付，保留 K8s 演进路径
- 数据库兼容：同一业务域支持 PostgreSQL 与 MongoDB 二选一运行
- 目标：快速交付、可长期演进、可商业化扩展

---

## 阶段 1：基础骨架与工程规范

### 目标

建立可长期维护的 Monorepo 骨架，保证后续迭代不会“推倒重来”。

### 任务清单

- [x] 初始化 Monorepo（pnpm + Turborepo）
- [x] 建立目录结构：apps、packages、infra、docs
- [x] 配置共享工程规范：ESLint、Prettier、Commitlint、Husky
- [x] 建立共享配置包：packages/config
- [x] 配置统一环境变量规范（.env.example + 校验）
- [x] 打通一键启动脚本（dev、build、test、lint）

### 产出物

- 可运行的项目骨架
- 统一编码和提交规范
- 新成员可复制环境并启动

### 阶段完成标准

- [ ] `pnpm i && pnpm dev` 可同时拉起 web/api
- [ ] 提交前自动执行 lint 与基础检查
- [ ] README 启动说明可被严格复现

---

## 阶段 2：后端分层与双数据库兼容底座

### 目标

实现“业务逻辑不依赖具体数据库”的架构底座，支持 PostgreSQL / MongoDB 切换。

### 任务清单

- [x] 建立后端分层：接口层、应用层、领域层、基础设施层
- [x] 定义仓储接口（Repository Port）与领域模型
- [x] 实现 PostgreSQL 适配器（Prisma）
- [x] 实现 MongoDB 适配器（Mongoose 或 Prisma Mongo）
- [x] 通过 `DB_PROVIDER` 环境变量选择数据实现
- [x] 建立迁移与初始化策略：
- [x] PostgreSQL 使用 Prisma Migrate
- [x] MongoDB 使用 seed + 索引初始化脚本
- [x] 建立集成测试基线（双数据库各跑一套）

### 产出物

- 不改业务代码切换数据库的能力
- 可验证的一致性测试用例

### 阶段完成标准

- [x] 同一组 API 在 PostgreSQL 模式通过测试
- [x] 同一组 API 在 MongoDB 模式通过测试
- [x] 业务服务层无任何 ORM 特定类型泄漏

## 阶段 3：认证、权限、多租户核心闭环

### 目标

形成企业可用的基础能力闭环：登录鉴权、组织关系、权限控制、多租户隔离。

### 任务清单

- [x] 实现 JWT + Refresh Token 鉴权链路
- [x] 建立用户、团队、角色、权限模型
- [x] 实现 RBAC（菜单级/页面级/按钮级）
- [x] 数据级权限策略（按租户、部门、创建人）
- [x] 建立多租户上下文注入中间件
- [x] 租户数据隔离（查询自动带租户条件）
- [x] 前端实现动态路由与权限指令

### 产出物

- 登录可用的后台模板
- 最小企业组织与权限能力

### 阶段完成标准

- [x] 超级管理员可创建租户、组织、角色并分配权限
- [x] 非授权用户无法访问受控路由/API
- [x] 跨租户数据查询严格隔离

## 阶段 4：业务基础模块与异步能力

### 目标

补齐企业项目高复用的“通用业务中台模块”。

### 任务清单

- [x] 通知中心：邮件/短信/站内信统一抽象
- [x] 文件中心：本地存储与对象存储（S3/OSS）可切换
- [x] 审计中心：登录日志、操作日志、敏感操作留痕
- [x] 任务中心：BullMQ 队列、重试、死信、补偿
- [x] 开放平台基础：API Key、签名校验、调用日志
- [x] SDK 生成链路：OpenAPI -> packages/sdk

### 产出物

- 可复用的企业基础模块
- 对外调用能力最小闭环

### 阶段完成标准

- [x] 通知、文件、审计、任务模块均有 API + 管理页面
- [x] API Key 可独立鉴权并记录调用日志
- [x] SDK 可供前端或第三方服务调用

## 阶段 5：可观测性、安全门禁与质量体系

### 目标

让系统具备可运维、可审计、可回滚的上线能力。

### 任务清单

- [x] 接入 OpenTelemetry（trace/span）
- [x] 暴露 Prometheus 指标，配置 Grafana 看板
- [x] 接入 Sentry（前后端异常）
- [x] 测试分层：单测、集成、端到端
- [x] 覆盖率阈值与 PR 强制检查
- [x] 安全扫描：依赖漏洞、secret、镜像扫描
- [x] 数据安全：字段脱敏、敏感字段加密存储

### 产出物

- 监控告警体系
- 可持续质量门禁

### 阶段完成标准

- [x] 核心链路有自动化测试并稳定通过
- [x] 任一关键错误可通过日志+trace定位
- [x] 高危漏洞/secret 泄漏可在 CI 阻断

## 单人开发执行策略（贯穿全程）

- [ ] 每阶段只设 1 个主目标，避免并行过多导致上下文切换
- [ ] 每阶段结束必须完成“文档 + 脚本 + 最小测试”三件套
- [ ] 采用 trunk-based + 短分支，PR 小步提交
- [ ] 先实现可用，再做可扩展；先保证主链路，再覆盖边角场景
- [ ] 预留 20% 缓冲处理返工、线上问题与文档补齐

---

## 最终验收清单（企业可用标准）

- [ ] 新项目初始化后可一键启动并登录
- [ ] PostgreSQL / MongoDB 任一模式可独立运行
- [ ] 核心链路自动化测试在 CI 稳定通过
- [ ] 关键操作均有审计记录与可追踪上下文
- [ ] 可一键部署测试环境并完成回滚演练
- [ ] 文档完整覆盖开发、部署、运维、二开、排障
